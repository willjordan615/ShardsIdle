== DELIVERED THIS SESSION ==

backend/database.js
- Migration adds three new columns to characters table: idleChallengeId TEXT, idlePartyIds TEXT, idleStartedAt INTEGER.
- Three new helpers exported: setIdleSession(characterId, challengeId, partyIds), getIdleSession(characterId), clearIdleSession(characterId).

backend/routes/combat.js
- POST /idle/start — stamps idle session on the primary owned character when loop activates.
- POST /idle/stop — clears idle session when loop is cancelled.
- POST /idle/collect — core offline system. Reads session, caps elapsed at 24hrs, divides by 60s minimum cycle to get combat count, runs full engine.runCombat() loop with live character state persisted across runs. Applies XP, skill merges, loot, drop chance ramp, lastSuccessfulChallengeId per run. Returns aggregate summary including skillXP deltas, newDiscoveries, lootGained, xpGained, wins/losses.
- getXPToNextLevel() and _xpThresholdForLevel() helpers mirror client-side formulas exactly.
- Bug fixes from earlier: getCombatEngine() renamed to initializeCombatEngine() (the actual function name), inline require() calls replaced with already-imported fs/path.

js/combat-system.js
- _notifyIdleStart() and _notifyIdleStop() fire-and-forget helpers.
- startCombat() calls _notifyIdleStart() each time the loop fires (re-stamps session timestamp).
- requestLoopExit() calls _notifyIdleStop().
- cancelLoopExit() calls _notifyIdleStart() to re-stamp since player returned to loop.

js/combat-log.js
- cancelAutoRestart() calls _notifyIdleStop().
- sleep() during combatSkipPlayback now compresses delays proportionally so catch-up cycles take at least 60 seconds rather than resolving instantly. Scale = min(1.0, 60000 / watchedCombatMs).
- _combatHiddenMidCycle now resets to false in the visibilitychange show handler so next cycle starts clean.
- _watchedCombatMs guard updated: skip-playback cycles no longer update the calibration timer (they complete in ~0ms and would corrupt the estimator).
- Speed display now shows seconds for sub-90s combats ("~45s") and minutes for longer ones, with Math.max(1,...) to avoid "~0 Min".

js/offline-summary.js (new file)
- checkOfflineProgress() — called from init block after initializeGame(). Iterates owned characters, calls /idle/collect on each. First one with hadSession:true triggers the summary screen. Returns true if shown, false otherwise.
- _renderOfflineSummary() — full summary screen render. Hero header with avatar portrait (uses window.AVATARS.renderCardBg), CSS shimmer sweep, floating gold particles, stat cards (time away, combats, win rate). Character XP bars using rm-xp-row/rm-bar--char markup (inherits combat-rewards CSS). Skill XP bars using rm-skill-row markup (identical to combat modal). Discoveries section with gold styling. Loot list sorted by rarity. Auto-sold duplicate gold/dust summary.
- _animateOfflineBars() — runs after DOM settles. Animates char XP bars first, then skill bars staggered, particle bursts on level-ups and discoveries. Local copies of animateBar and spawnParticles (named _osAnimateBar, _osSpawnParticles) — no dependency on combat-rewards.js IIFE scope.
- _dismissOfflineSummary(resumeLoop) — Return to Roster path calls renderRoster() then showScreen('roster'). Resume Loop path loads character from DB, sets currentState, calls startCombat().
- _viewCharacterFromSummary() — See Character button. Calls renderRoster() then showCharacterDetail() with primary character ID. No loop started.
- window._offlineSummaryChallengeId and window._offlineSummaryPrimaryChar stashed on render for use by dismiss/resume/view paths.

js/inventory-system.js
- _sortInventoryItems(items, sortKey) — sorts within slot groups by: type (alpha by def.type, damage desc within type), damage (total dmg1-4 desc), stats (sum of all stat fields desc), proc (has onhit_skillid_1 or effect_skillid), newest (acquiredAt desc). Tiebreaker is always newest.
- _invSortKey module-level variable persists across re-renders, defaults to 'type'.
- _renderGearModal() accepts optional third param sortKey.
- Sort bar renders between slot tabs and item list using inv-tab/inv-tab--active styling.
- _procLine(def) helper — iterates onhit_skillid_1/2/3, looks up skill name from gameData.skills, formats as "↯ Burn (15%)". Renders in light blue (#a0d4ff) on both equipped and inventory rows. Uses ↯ (U+21AF) not emoji.
