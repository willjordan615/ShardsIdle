Map Architecture
Screen Structure
html<div id="map" class="screen" style="padding:0;">
  <div class="nav-buttons">← Back to Character | The Silver Vale — Field Codex</div>
  <div id="mapWrap">
    <img id="mapBgImg" src="/assets/map.png">
    <div id="mapNodesLayer">  ← nodes appended here by mapBuildNodes
    <!-- Legend bar pinned to bottom (position:absolute) -->
  </div>
</div>
After mapInitPanZoom() runs, a #mapInner div is created inside #mapWrap and the image + nodes layer are moved inside it. #mapInner is sized to the image's natural aspect ratio (1408×768) and the transform is applied to it for pan/zoom.
Key Functions (all inline in index.html)

window.initMapScreen(characterId) — async, fetches character level, calls mapBuildNodes then mapInitPanZoom
mapBuildNodes(level) — creates/updates node divs, sets state, pip colors, label text
mapGetState(c, lvl) — four tiers: available (within 7), near (within 14), approaching (within 21), locked
mapStartScramble(id, name, el) — one-shot reveal animation for newly available nodes
mapStartScrambleLoop(id, len, el) — continuous glyph cycling for near nodes
mapOpenModal(c) — populates and shows modal, pulls full data from window.gameData.challenges
mapInitPanZoom() — creates #mapInner, sizes to aspect ratio, wires all pan/zoom/touch events

Node States
StateTriggerPipLabelavailablewithin 7 levels of minLvlGold→red gradient + difficulty numberFull namenearwithin 14 levelsMuted red + difficulty numberHalf-length scramble loopapproachingwithin 21 levelsDim red border, no number~40% rune fragmentlockedbeyond 21 levelsDark, no number2 glyphs
Pip Color Gradient
Available nodes use a continuous interpolation from gold rgb(212,175,55) at minLvl to dark red rgb(160,40,40) at 21 levels below. t = (c.minLvl - level) / 21, clamped 0–1. Same gradient applied (with alpha) to near and approaching pips.
ID Mismatch Workaround
MAP_CHALLENGES uses IDs like whispering_willow_shrine. window.gameData.challenges uses challenge_whispering_willow_shrine. The modal's Select Challenge button resolves this:
jswindow.gameData.challenges.find(c => c.id === mapCurrentChallenge.id || c.id === 'challenge_' + mapCurrentChallenge.id)

What Still Needs Doing
1. Viewport / Map Height (the unresolved problem)
This is the main outstanding issue. The map needs to sit between the global header (#headerTopBar) and fill the remaining viewport height below it, with the legend bar overlaid at the bottom of #mapWrap. The nav buttons row (Back to Character) should sit between the header and the map, like every other screen.
The problem: #mapWrap has no explicit height, and the flex chain through .container → #map → #mapWrap doesn't resolve because .container has no fixed height. Multiple approaches were tried:

position:fixed; inset:0 on #map — map filled viewport but covered header
height:calc(100vh - Xpx) on #mapWrap — brittle, pixel values don't match header height reliably
Flex chain with height:100% on body/html/container — map had no height

Recommended approach for next session: Set height: calc(100vh - var(--header-height)) on #mapWrap, where --header-height is a CSS variable set to the actual rendered height of #headerTopBar. Or simpler: measure #headerTopBar + .nav-buttons offsetHeight in mapInitPanZoom and set #mapWrap height explicitly via JS before sizing #mapInner.
2. Name Truncation at Default Zoom
Labels use max-width that scales with zoom (168 * scalepx). At default zoom names like "Whispering Willow Shrine" still truncate. Either increase the base 168px or reduce font size slightly.
