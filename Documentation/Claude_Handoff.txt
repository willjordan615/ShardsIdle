js/skill-tree.js — NEW FILE

Skill web modal accessible from a "⬡ Skill Web" button next to the Skills section title
  on the character detail screen. SVG canvas, pan by drag, zoom by scroll/pinch.

Node visibility rules:
  owned (gold, solid) — skillLevel >= 1 or intrinsic
  discovering (dim gold, XP bar) — skillLevel 0, both parents known
  reachable (dark blue, dashed) — one or both parents known, child not yet in skills
  hidden — zero parents known, not shown at all

Name reveal rule: reachable nodes show "???" until any known parent reaches level 3.
  At level 3+ the name reveals. Missing parent shown as "???" in tooltip, never named.
  Tooltip for unrevealed nodes says "Something stirs in the dark."

Click a node to highlight its children and dim everything else. Click again or click
  background to deselect. Selection resets on each modal open.

Character data fetched async via getCharacter() — loading state ("Reading the paths…")
  shows immediately, SVG renders once data arrives.

SVG text inherits font via CSS rule (#skillTreeModal text). SVG rect fills use CSS
  classes (st-node-owned, st-node-discovering, st-node-reachable) so they respond to
  the window color theme setting.

js/combat-rewards.js — NEW FILE

Animated victory/defeat modal. Replaces the static result modal content entirely.
  Called via window.animateCombatRewards(payload) from applyCombatRewards in combat-log.js.

Animation sequence:
  1. Title (VICTORY / DEFEATED / RETREATED) shown immediately on modal open as loading state
     with a gold shimmer pulse beneath it while rewards compute server-side.
  2. On animateCombatRewards call: loot items cascade in with CSS stagger by animation-delay.
     Gear items shake on arrival — amplitude scales by rarity (common: small nudge,
     uncommon: wobble, rare: scale+shake, legendary: brightness flash + aggressive shake).
     Consumables flow in quickly without shake.
  3. Character XP bar sweeps from pre-combat to post-combat value. Flashes + particle burst
     on level-up.
  4. Skill XP bars animate from before to after. Gold bars for discovering skills, blue for
     mastery. Flash and particle burst on level boundary crossing.
  5. Footer (countdown/buttons) fades in last.

Particle system: absolute-positioned divs, CSS transition, auto-removed after life expires.
  Color and count scale by rarity and event type.

Payload shape passed from applyCombatRewards:
  { result, loot: [{name, rarity, isGear}], charXP: [{...}], skillXP: {...}, newUnlocks: [...] }
